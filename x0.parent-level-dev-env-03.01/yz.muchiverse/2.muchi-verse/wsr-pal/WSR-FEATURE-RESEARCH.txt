PART 2: TRANSACTION LOG - WHAT THIS MEANS
================================================================================

SIMPLE EXAMPLE:

You decide to buy 100 shares of ORB at $50 each.
System records this trade.

Then next week, you buy another 50 shares at $52 each.
System records this second trade too.

TRANSACTION LOG is just a file that remembers EVERY TRADE:

  File: pieces/player_you/transactions.txt

  ORB|buy|100|50.00|2026-07-17
  AFL|buy|200|25.50|2026-07-18
  ORB|buy|50|52.00|2026-07-18
  ORB|sell|30|60.00|2026-07-19

Each line = one trade
Format: TICKER|ACTION|SHARES|PRICE|DATE

WHY DO WE NEED THIS?

To show in Portfolio:
  ORB: You own 120 shares total (100+50-30)
       Average buy price: $50.92 per share
       Current price: $116.00 per share
       Total gain: +$7,833

Without the transaction log, we can't calculate "average buy price" or "total gain".


PART 3: HOW AI TRADING WORKS (IT'S REAL)
================================================================================

Original WSR game has an analysis_loop.c that calculates stock prices.
wsr-pal PORTED this real logic. Here's how it works:

FUNDAMENTAL VALUE CALCULATION:

  For every stock, the AI calculates what it SHOULD be worth:

  1. Start with: book_value / shares_outstanding
     (This is the company's net worth per share)

  2. Adjust for market size:
     Large companies are more stable → price multiplier 1.05
     Tiny companies are volatile → price multiplier 0.8

  3. Adjust for debt level:
     High debt (leverage) = risky → LOWER stock price
     Low debt = safe → HIGHER stock price

  4. Adjust for "risk bias":
     Each company has a risk profile (1-100 scale)
     Aggressive company (risk=90) → higher price multiplier
     Conservative company (risk=10) → lower price multiplier

  5. Blend with current price:
     70% from calculated fundamentals
     30% from what it's trading at now
     (Prevents wild swings, maintains momentum)

AI TRADING DECISION:

  IF current_price < fundamental_value:
     BUY (stock is undervalued)

  IF current_price > fundamental_value:
     SELL (stock is overvalued)

PROOF IT WORKS:

  Test from 2026-07-17:
  - ORB stock was $167.73
  - After 5 ticks, it moved to $115.04 (-31%)
  - This happened because fundamentals changed (not random)
  - Financial news correctly reported: "CMZ: -99.38% biggest mover"
  - Different AI traders made different decisions based on their own risk profiles
  - Result: realistic price discovery through competing analysis

IMPORTANT: This is NOT vaporware. The original game had bugs where AI traders
made byte-identical copies of each other (meaningless). wsr-pal uses REAL
analysis_loop.c ported to our system. Each corporation's decision_mode=1
(weighted) makes genuine fundamental-value trades.

FEATURE 3: SCREENER / RESEARCH REPORT (4-5 hours)
================================================================================

WHAT IT DOES:
  1. Scans ALL corporations in pieces/corp_*/
  2. Calculates fundamental value for each stock
  3. Compares to current price to find opportunities
  4. Lets user filter by criteria:
     - Market cap (small/mid/large)
     - P/E ratio (price vs fundamentals)
     - Leverage (debt level)
     - % price change (52-week movers)
  5. Displays sorted list for user to browse

EXAMPLE SCREENER OUTPUT:

  RESEARCH REPORT - STOCKS UNDER FUNDAMENTAL VALUE

  TICKER | PRICE | FUNDAMENTAL | UPSIDE | P/E | LEVERAGE | YIELD
  -------|-------|-------------|--------|-----|----------|-------
   ZZZ   | $45   |    $78.50   | +74%   | 0.57| 0.25     | 2.1%
   XYZ   | $32   |    $55.00   | +72%   | 0.58| 0.18     | 1.8%
   ABC   | $120  |    $95.00   | -21%   | 1.26| 0.95     | 0.5%

  User can select a stock to see full details or buy immediately


FILES TO CREATE:

  ops/+x/screener.+x
  - NEW OPERATION: calculates fundamentals, filters, sorts
  - Input: filter criteria (market cap, P/E, etc.)
  - Output: sorted table of stocks matching criteria

  pieces/screener_menu/piece.pdl
  - NEW MENU: filter options
    [>] 1. [Undervalued Stocks]
    [ ] 2. [High Growth]
    [ ] 3. [Dividend Aristocrats]
    [ ] 4. [Small Cap Gems]
    [ ] 5. [Back to Main Menu]

CALCULATION (from analysis_loop.c):

  For each stock:
    book_value_per_share = book_value / shares_outstanding
    leverage_factor = 1.0 (adjusted if debt_to_equity > 1.0)
    market_cap_multiplier = 1.0 (adjusted based on size)
    risk_bias = 0.5 to 1.5 (from corp's risk profile)

    fundamental_value = book_value_per_share
                      * market_cap_multiplier
                      * leverage_factor
                      * risk_bias

    upside = (fundamental_value - current_price) / current_price * 100%
    p_e_ratio = current_price / (book_value_per_share * risk_bias)

  4. See list of stocks trading below fundamental value
  5. Select one → view details
  6. Buy from research screen
  7. Check portfolio → verify buy recorded

PART 6: DATA STRUCTURES NEEDED
================================================================================

TRANSACTIONS.TXT FORMAT:

  File: pieces/player_you/transactions.txt

  One trade per line:
  TICKER|BUY_OR_SELL|SHARES|PRICE|DATE

  Example:
    ORB|buy|100|50.00|2026-07-17
    AFL|buy|200|25.50|2026-07-18
    ORB|buy|50|52.00|2026-07-18
    ORB|sell|30|60.00|2026-07-19

  Current holdings = SUM(buy shares) - SUM(sell shares) per ticker
  Average cost basis = SUM(buy_shares * buy_price) / current_holdings


WHO WRITES TRANSACTIONS.TXT?

  player_trade.c (existing operation):
    When player buys/sells, it updates both:
    - pieces/player_you/holdings.txt (current snapshot)
    - pieces/player_you/transactions.txt (NEW: append new trade)
