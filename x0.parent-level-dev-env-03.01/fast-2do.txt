## Modularity Rule

Do not hardcode validation-project bodies, scene records, ops, cube data, map data, widget data, or emoji extrusion data in `wraith-alpha_manager.c`.

Current file-owned Wraith project bridge:
- `session/wraith_body.txt` controls the body/audit text shown inside the Wraith window.
- `session/scene.objects.pdl` controls project semantic objects imported into the frame.
- project assets remain under that project: `pieces/**`, `maps/**`, `assets/**`, `layouts/**`.

Wraith Alpha may discover and host projects. Individual Wraith projects must own their own layouts/state/scene/pieces/maps/assets. If it is not in a file or receipt, it is not project truth.

## Required Audit Workflow

Before changing visual layout:
1. Read `projects/wraith-alpha/session/rgb/current_frame.receipt.pdl`.
2. Read `projects/wraith-alpha/session/rgb/gl_display.receipt.pdl`.
3. Confirm `render_checksum_fnv1a64` matches `loaded_rgba_checksum_fnv1a64`.
4. Inspect object `text_px_*`, `hit_px_*`, and `focus_px_*`.
5. For cube/map/emoji render work, inspect `SECTION | PRIMITIVES | SOURCE_AND_CONVERSION_AUDIT`.
6. Only then change coordinates, projection, or styling.

Before changing mouse behavior:
1. Read `projects/wraith-alpha/session/rgb/gl_input.receipt.pdl`.
2. Check raw mouse, texture viewport, mapped texture coordinates, and `inside_texture`.
3. Do not change ASCII mouse offsets to fix GL fullscreen issues.

## Architecture Reminders

- One scene, two presenters.
- ASCII is an audit/debug projection.
- GL is a graphical presentation projection.
- `wraith_gl.c` should stay thin.
- `wraith_rgb_daemon.c` should own RGB conversion.
- the manager/session artifacts should own Wraith window/project truth.
- `${game_map}`, `INTERACT`, and `is_map_control` are the uniform names for now.
- `piececraft-wraith` and `chtmgl-wraith` are new Wraith project names; do not silently reuse source project names unless the code/project is truly the same.
- 2D-to-3D conversion is special to `${game_map}` surfaces only:
  - cube/entity: `pieces/<id>/artifact.txt` z-slice piece.
  - world/map: `maps/map_01_z*.txt` plus tile registry/extrude.
  - image/emoji: `pieces/<name>/voxels_<res>.csv` RGBA extrusion.
