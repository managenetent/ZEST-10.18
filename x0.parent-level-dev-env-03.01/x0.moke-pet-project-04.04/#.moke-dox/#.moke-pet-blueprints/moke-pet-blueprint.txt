# Moke-Pet (Lizard Bot) Architectural Blueprint

## 1. Core Philosophy
The Moke-Pet system follows the **Container-First, Sovereign-Piece** model with **Dynamic Resource Discovery**.
- **Tank = World/Map Container**: The primary boundary.
- **Dynamic Identification**: Lizards do not have hardcoded food paths. They scan the container for any piece with `type: food` in its PDL/state.
- **Sovereign Actions**: 
    - `scan.+x`: Perceptual Op; populates `memory/observations.txt` with nearby pieces.
    - `eat.+x`: Action Op; targets a specific piece ID from observations.
    - `attack.+x`: Action Op; targets another lizard.

## 3. Biological & Survival Mechanics (Evolution)
- **Metabolism**: Every turn increases `hunger`. If `hunger > threshold`, the entity dies.
- **Consumption**: `eat` moves food pieces to `stomach/`, removing them from the enclosure.
- **Predation (Lethal Combat)**: 
  - `attack` reduces enemy `hp`. 
  - On death (`hp <= 0`), the entity directory is renamed (e.g., `liz_char` -> `food_char`) and its `type` is changed to `food`, making it edible.
- **Reproduction (Mating)**: 
  - If two lizards occupy the same space/condition, they can trigger a `mate` Op.
  - A new sovereign entity directory is created and added to the manager's turn rotation.

## 5. Controller Compatibility Note
- The same engine may later accept a controller that pauses, steps, or takes one-turn manual control.
- If no controller is attached, the simulation remains fully autonomous.
- If a controller selects AI, it should default to one step unless `resume` is active.
- The same project root should own the enclosure directory whether launched autonomously or through the GUI.
- Controllers may live at the project top level or under `controllers/`, but the executable should still be located in a `+x/` directory.
- `xo-pet-launch.sh` is the descriptive entrypoint for this bundle.
- `button.sh` is only a legacy wrapper kept during the rename.
- `xo-pet-launch.sh` should accept `auto` and `controller <uid_pair>` so the user never has to specify an executable name manually.
- If `controller` is used without a uid pair, `xo-pet-launch.sh` should auto-generate one for the session.
