# Lab Plan D16: Model Deconstruction and Visualization

This document outlines the plan for two new programs to be created in the `lab/` directory. The goal of these programs is to deconstruct the knowledge learned by our layer-centric transformer and visualize it in a token-centric way for educational and data science purposes.

---

## Program 1: `probe_model.c` (The Model Prober/Converter)

**Purpose:**
To "probe" the trained layer-centric model (using the `.weights` file) and generate a new, token-centric data file. This file will contain rich vector representations for each token in the vocabulary, effectively projecting the model's distributed knowledge onto a per-token basis.

**Command-line Interface:**
`./+x/probe_model.+x <vocab_file> <weights_file_path_base> <output_probe_file>`
- Example: `./+x/probe_model.+x curriculum/guess_the_word/guess_the_word.txt curriculum/guess_the_word/guess_the_word.txt lab/probe_data.txt`

**Process:**
1.  **Load Model:** The program will start by loading the full model definition, including vocabulary, embeddings, and the trained layer weights from the specified files. It will reuse the `SimpleModel`, `SimpleVocab`, and `RunState` structs.
2.  **Iterate and Probe:** It will loop through every single token in the vocabulary.
3.  **Forward Pass:** For each token, it will perform a single `transformer_forward()` pass with that token as the input at `position = 0`. This context-free forward pass will generate a set of activations that represent how the model processes that specific token.
4.  **Capture Activations:** After the forward pass, the program will capture key vectors from the `RunState` struct:
    - The final output vector `state.x` (the token's embedding after being processed by all layers).
    - The query vector `state.q` (from the final layer).
    - The key vector `k` (from the final layer's KV cache at `pos=0`).
5.  **Save Probe Data:** The captured vectors will be written to the `<output_probe_file>` in a human-readable text format.

**Output Format (`probe_data.txt`):**
The output will be a space-delimited file. Each line will correspond to one token from the vocabulary.
- **Format:** `Token_String Word_ID [Final_Embedding_Vector] [Query_Vector] [Key_Vector]`
- **Example:**
  ```
  the 1 0.111 0.222 ... 0.333 ... 0.444 ...
  a 2 -0.45 0.81 ... -0.15 ... 0.99 ...
  ```

## Program 2: `visualize_attention.c` (The Probe Data Analyzer)

**Purpose:**
To analyze the `probe_data.txt` file generated by `probe_model.c`. This program will not use the transformer model directly, but will instead operate on the "deconstructed" per-token vector data to glean insights and even simulate generation.

**Command-line Interface:**
`./+x/visualize_attention.+x <probe_data_file> <command> [args...]`

**Features & Commands:**

1.  **`similarity <word1> <word2>`**
    - **Action:** Loads the `probe_data.txt` file, finds the "Final_Embedding" vectors for `<word1>` and `<word2>`, calculates the cosine similarity between them, and prints the score.
    - **Purpose:** To quantitatively measure the semantic similarity between words as understood by the model.

2.  **`analogy <word1> <word2> <word3>`**
    - **Action:** Implements the classic `vec(word1) - vec(word2) + vec(word3)` analogy test. It finds the word in the vocabulary whose "Final_Embedding" vector is closest (by cosine similarity) to the resulting vector.
    - **Purpose:** To test if the model's learned vector space captures complex semantic relationships.

3.  **`generate <start_word> <num_tokens>`**
    - **Action:** Simulates text generation using only the deconstructed Query and Key vectors from the probe file.
    - **Algorithm:**
        1. Start with `<start_word>`. Its Query vector is `Q_current`.
        2. To find the next word, iterate through all words in the vocabulary, calculating a pseudo-attention score: `score = dot_product(Q_current, K_i)` where `K_i` is the Key vector of word `i`.
        3. Apply a softmax function to all scores to get a probability distribution.
        4. Sample the next word from this distribution.
        5. The newly sampled word's Query vector becomes the new `Q_current`.
        6. Repeat for `<num_tokens>`.
    - **Purpose:** To explore if a generative process can be simulated from the "deconstructed" attention components alone.

**Implementation Notes:**
- These two programs will be self-contained within the `lab/` directory.
- They will require copies of the necessary data structures (`SimpleModel`, etc.) and utility functions from `generation_module.c`. A shared header file might be created later to improve code reuse.
