## PARITY GAP #2: KEYBOARD INPUT NOT UPDATING DISPLAY
================================================================================

### Linux Behavior (PERFECT)

**Flow:**
```
1. User presses arrow key
2. keyboard_thread_func() reads key via termios
3. writeCommand() writes to pieces/keyboard/history.txt
4. writeCommand() writes to pieces/master_ledger/master_ledger.txt
5. chtpm_parser reads history, processes input
6. chtpm_parser updates pieces/display/current_frame.txt
7. chtpm_parser touches pieces/display/renderer_pulse.txt
8. render_thread_func() detects pulse change
9. render_thread_func() prints new frame
```

**File:** `pieces/chtpm/plugins/orchestrator.c`

**Keyboard Thread (Linux):**
```c
void enableRawMode() {
#ifndef _WIN32
    tty_fd = STDIN_FILENO;
    tcgetattr(tty_fd, &orig_termios);
    atexit(disableRawMode);
    struct termios raw = orig_termios;
    raw.c_iflag &= ~(BRKINT | ICRNL | INPCK | ISTRIP | IXON);
    raw.c_cflag |= (CS8);
    raw.c_lflag &= ~(ECHO | ICANON | IEXTEN | ISIG);
    raw.c_cc[VMIN] = 0;
    raw.c_cc[VTIME] = 1;  /* 0.1 second timeout */
    tcsetattr(tty_fd, TCSAFLUSH, &raw);
#endif
}

int readKey() {
#ifndef _WIN32
    int nread;
    char c;
    while ((nread = read(tty_fd, &c, 1)) != 1) {
        if (nread == -1 && errno != EAGAIN) return -1;
        usleep(10000);
    }
    // ... handle escape sequences for arrows ...
#endif
}

void* keyboard_thread_func(void* arg) {
    enableRawMode();
    while (!should_exit) {
        int c = readKey();
        if (c == -1) continue;
        if (c == 3) { should_exit = 1; break; }  // Ctrl+C
        writeCommand(c);
        usleep(10000);
    }
    return NULL;
}
```

### Windows Behavior (BROKEN)

**Keyboard Thread (Windows):**
```c
void enableRawMode() {
#else
    h_console_input = GetStdHandle(STD_INPUT_HANDLE);
    if (h_console_input != INVALID_HANDLE_VALUE) {
        GetConsoleMode(h_console_input, &orig_console_mode);
        DWORD mode = orig_console_mode & ~(ENABLE_LINE_INPUT | ENABLE_ECHO_INPUT);
        SetConsoleMode(h_console_input, mode);
    }
#endif
}

int readKey() {
#else
    if (_kbhit()) {
        c = (char)_getch();
    } else {
        Sleep(100);  /* 100ms */
        return -1;
    }
    // ... handle escape sequences ...
#endif
}
```

**Potential Issues:**

1. **Sleep timing difference:**
   - Linux: `usleep(10000)` = 10ms in readKey loop
   - Windows: `Sleep(100)` = 100ms in readKey loop
   - Impact: Windows may miss rapid key presses

2. **Console mode flags:**
   - Linux: Disables ICANON, ECHO, ISIG, IEXTEN
   - Windows: Disables ENABLE_LINE_INPUT, ENABLE_ECHO_INPUT
   - Missing: ENABLE_PROCESSED_INPUT (Windows equivalent of ISIG?)

3. **Escape sequence handling:**
   - Linux: Reads 3 bytes for escape sequences (`\x1b[A` = up arrow)
   - Windows: Uses `_getch()` which returns scan codes (72=up, 80=down)
   - Check: Are both writing same key codes to history?

4. **writeCommand() output:**
   - Check: Does Windows write identical format to history.txt?

## PARITY GAP #3: CTRL+C DOESN'T KILL GL WINDOW
================================================================================

### Linux Behavior (PERFECT)

**File:** `pieces/chtpm/plugins/orchestrator.c`

**Code:**
```c
void handle_sigint(int sig) {
    should_exit = 1;
    disableRawMode();
    fprintf(stderr, "\n[orchestrator] Received signal %d...\n", sig);

#ifndef _WIN32
    /* Kill all processes in the process group */
    kill(0, SIGTERM);  // Sends to ALL processes in group

    /* Specifically kill GL renderer */
    system("pkill -9 -f 'gl_renderer' 2>/dev/null");

    /* Run cleanup script */
    system("./kill_all.sh > /dev/null 2>&1");
#else
    system("powershell.exe -File ./kill_all.ps1 > $null 2>&1");
#endif

    exit(0);
}
```

**Why it works:**
- `kill(0, SIGTERM)` sends signal to entire process group
- GL renderer receives SIGTERM and exits
- `pkill -9` provides backup kill
- `kill_all.sh` cleans up stragglers

### Windows Behavior (BROKEN)

**Code:**
```c
#else
    system("powershell.exe -File ./kill_all.ps1 > $null 2>&1");
#endif
```

**Problems:**

1. **No process group kill:** Windows doesn't have `kill(0, SIGTERM)`
2. **No explicit GL renderer kill:** Missing `pkill` equivalent
3. **powershell script may be slow:** Takes time to start, GL may exit first
4. **_spawnl(_P_DETACH):** Fully detached processes, not in job object

### Fix Required

**Option A: Add explicit taskkill commands**

```c
#else
    /* Kill GL renderer explicitly (Windows equivalent of pkill) */
    system("taskkill /F /IM gl_renderer.+x 2>nul");
    system("taskkill /F /IM orchestrator.+x 2>nul");
    system("taskkill /F /IM renderer.+x 2>nul");
    system("taskkill /F /IM chtpm_parser.+x 2>nul");
    system("taskkill /F /IM clock_daemon.+x 2>nul");
    
    /* Then run full cleanup */
    system("powershell.exe -File ./kill_all.ps1 > $null 2>&1");
#endif
```

**Option B: Use Windows Job Objects (advanced)**

Create a job object, assign all child processes, kill job on exit.
(This is the Windows equivalent of Unix process groups.)

**Option C: Use PowerShell to kill process tree**

```c
#else
    /* Kill entire process tree */
    system("powershell -Command \"Get-Process | Where-Object {$_.Path -like '*TPMOS*'} | Stop-Process -Force\" 2>$null");
    system("powershell.exe -File ./kill_all.ps1 > $null 2>&1");
#endif
```
