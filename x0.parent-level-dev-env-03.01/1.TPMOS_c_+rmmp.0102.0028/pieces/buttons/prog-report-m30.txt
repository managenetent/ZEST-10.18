## Architecture: How Spawn Works

### Linux (fork + exec)
```c
pid_t p = fork();
if (p == 0) {
    // Child process
    execl(path, path, args, NULL);
} else if (p > 0) {
    // Parent: wait for child
    waitpid(p, NULL, 0);
}
```

### Windows (_spawn)
```c
int pid = _spawnl(_P_DETACH, path, path, args, NULL);
if (pid > 0) {
    // Child spawned and running detached
    // Parent continues immediately
}
```

### Key Differences
| Aspect | Linux fork() | Windows _spawn() |
|--------|-------------|------------------|
| Process model | Copy parent, then exec | Direct exec |
| I/O redirection | Easy (freopen in child) | Hard (requires handles) |
| Wait for exit | waitpid() | _cwait() or none for detached |
| Detached | No (parent waits) | Yes (_P_DETACH) |

### How It Works

The batch files call shell scripts that:
1. Set `PATH` to include MSYS2 Mingw64 bin (for freeglut.dll, etc.)
2. Use bash to execute `.+x` files directly (bash doesn't care about extensions)

```bash
#!/bin/bash
export PATH=/c/msys64/mingw64/bin:/c/msys64/usr/bin:$PATH
./binary.+x  # Works because bash executes it directly
```
