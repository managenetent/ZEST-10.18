## 1. Exo-Sovereignty: The "External Operating" Model
The XO Bot is an **Exo-Bot**. Unlike future internal system bots, it is sovereign to its own directory tree and designed to operate *externally* to the project root.

### 1.1 Sovereign Memory
- The bot's data (Memories, Weights, PDL) lives inside `xo/bot5/pieces/`.
- This ensures the bot's cognition is portable and independent of the ASCII-OS's native piece tree.
- **Vision/Motor Interface**: It reaches *into* the project root (e.g., `pieces/display/current_frame.txt`) to observe, but stores its learning *externally*.

### 1.2 Long-Term Traversal
- Ultimately, XO bots will possess **Directory Traversal** abilities similar to human agents.
- They will be capable of existing and operating outside the `./` parent directory, managing multiple TPMOS instances or versioned workspaces autonomously.

---

## 2. Agent-Supervised Imitation Learning (ASIL)
- **Observation**: `record_session.+x` monitors the project streams.
- **External Memories**: Sequences are stored in `xo/bot5/pieces/memories/`.
- **Labeling**: Human agents rename these external folders to define goals.

---

## 3. RL Weight Training (Policy Layer)
- **Weights Piece**: `xo/bot5/pieces/weights/` stores probability matrices.
- **Autonomous Drive**: The bot uses these external weights to decide which inputs to inject into the internal `history.txt` file.
