# POST-FIX TESTING GUIDE - OP-ED RESTORATION
**Date:** 2026-03-25
**Purpose:** Systematic testing procedure to verify op-ed fix and prevent regressions
**Scope:** Black-box runtime testing + state integrity verification

---

## HOW TO USE THIS GUIDE

1. **Before Testing:** Apply the fix from `op-ed-fix.txt`
2. **During Testing:** Follow test sequence T1-T12 in order
3. **After Testing:** Record results in `#.testing/op-ed-test-results-YYYY-MM-DD.txt`

**Testing Principles:**
- Test existing behavior FIRST (regression baseline)
- Test new behavior SECOND (fix verification)
- Record evidence (frame snippets, state file contents)
- In no-git mode, create manual backups before risky changes
- After each test pass, publish frame-history report

---

### Step 1: Create Backup (MANDATORY)

## COMMON ISSUES AND TROUBLESHOOTING

### Issue: op-ed shows "PROJECT MANAGER" instead of "O P - E D   A D V A N C E D"
**Cause:** Missing app-level state initialization
**Fix:** Ensure `pieces/apps/op-ed/manager/state.txt` exists with correct content

### Issue: Xlector doesn't move
**Cause:** History file not being read or module not polling
**Fix:** Check `pieces/apps/player_app/history.txt` is being written and read

### Issue: Map doesn't display
**Cause:** Missing map file or wrong path
**Fix:** Verify `projects/op-ed/maps/map_01.txt` exists

### Issue: Glyph palette empty
**Cause:** Module not initializing glyph array
**Fix:** Check op-ed_module.c initialization code

### Issue: Exit doesn't work
**Cause:** Wrong href path or module not handling exit
**Fix:** Verify EXIT button href points to `pieces/chtpm/layouts/os.chtpm`
