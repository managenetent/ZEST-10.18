================================================================================
WHAT IS FONDU?
================================================================================

Fondu is the TPMOS project lifecycle manager. It:
- Moves projects between states (active ↔ archived ↔ installed)
- Registers ops so other projects can call them
- Maintains compile manifests
- Automates deployment for users

Location: `pieces/system/fondu/fondu.+x`

================================================================================
BASIC USAGE
================================================================================

./pieces/system/fondu/fondu.+x <command> [project_name]

Commands:
  --install <project>    - Deploy to production (read-only)
  --uninstall <app>      - Remove from production
  --archive <project>    - Move to trunk (source only)
  --restore <project>    - Restore from trunk
  --list                 - Show all projects
  --list-ops             - Show available ops
  --help                 - Show help

COMMON WORKFLOWS
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ WORKFLOW 1: Deploy a project to production                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  You finished developing a project and want a read-only production version: │
│                                                                             │
│  $ ./pieces/system/fondu/fondu.+x --install <project_name>                  │
│                                                                             │
│  Example:                                                                   │
│  $ ./pieces/system/fondu/fondu.+x --install fuzz-op                         │
│                                                                             │
│  What happens:                                                              │
│  1. Copies project to pieces/apps/installed/<project>/                      │
│  2. Sets read-only permissions (chmod 555)                                  │
│  3. Registers ops in pieces/os/ops_registry/<project>.txt                   │
│  4. Updates pieces/os/compiled_projects.txt                                 │
│  5. Generates ops_catalog.txt                                               │
│                                                                             │
│  Verify:                                                                    │
│  $ ./pieces/system/fondu/fondu.+x --list                                    │
│  → Should show: <project> (production)                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ WORKFLOW 2: Remove an installed app                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  You want to remove a production app (source stays intact):                 │
│                                                                             │
│  $ ./pieces/system/fondu/fondu.+x --uninstall <app_name>                    │
│                                                                             │
│  Example:                                                                   │
│  $ ./pieces/system/fondu/fondu.+x --uninstall fuzz-op                       │
│                                                                             │
│  What happens:                                                              │
│  1. Deletes pieces/apps/installed/<project>/                                │
│  2. Removes ops from pieces/os/ops_registry/                                │
│  3. Updates compiled_projects.txt                                           │
│  4. Regenerates ops_catalog.txt                                             │
│                                                                             │
│  Verify:                                                                    │
│  $ ls pieces/apps/installed/                                                │
│  → Should NOT show the app                                                  │
│  $ ls projects/<project>/                                                   │
│  → Source should STILL exist!                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ WORKFLOW 3: Archive a project (move to trunk)                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  You're not working on this project right now, want to exclude it from      │
│  compile cycles:                                                            │
│                                                                             │
│  $ ./pieces/system/fondu/fondu.+x --archive <project_name>                  │
│                                                                             │
│  Example:                                                                   │
│  $ ./pieces/system/fondu/fondu.+x --archive lsr                             │
│                                                                             │
│  What happens:                                                              │
│  1. Moves projects/<project>/ → projects/trunk/<project>/                   │
│  2. Removes from compiled_projects.txt                                      │
│  3. Unregisters ops                                                         │
│                                                                             │
│  Verify:                                                                    │
│  $ ./pieces/system/fondu/fondu.+x --list                                    │
│  → Should show under ARCHIVED, not ACTIVE                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ WORKFLOW 4: Restore a project from trunk                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  You want to work on an archived project again:                             │
│                                                                             │
│  $ ./pieces/system/fondu/fondu.+x --restore <project_name>                  │
│                                                                             │
│  Example:                                                                   │
│  $ ./pieces/system/fondu/fondu.+x --restore lsr                             │
│                                                                             │
│  What happens:                                                              │
│  1. Moves projects/trunk/<project>/ → projects/<project>/                   │
│  2. Adds back to compiled_projects.txt                                      │
│  3. Re-registers ops                                                        │
│                                                                             │
│  Verify:                                                                    │
│  $ ./pieces/system/fondu/fondu.+x --list                                    │
│  → Should show under ACTIVE, not ARCHIVED                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Permission denied" when uninstalling
Solution: Fondu should handle this automatically. If not, check that the
          remove_directory() function is setting chmod 755 before deletion.

Problem: Ops don't show in --list-ops
Solution: Check that ops_manifest.txt exists in project/ops/ and has correct
          format: op_name|description|args_format

Problem: Project disappears after uninstall
Solution: This is a BUG! Uninstall should only remove from installed/, never
          touch projects/. Check cmd_uninstall() in fondu.c.

Problem: Archive fails with "Directory not empty"
Solution: Check that trunk/<project>/ doesn't already exist. Fondu should
          handle this but may need cleanup.

================================================================================
FILES FONDU TOUCHES
================================================================================

Reads:
- projects/<project>/project.pdl
- projects/<project>/ops/ops_manifest.txt

Writes:
- pieces/apps/installed/<project>/ (created on install)
- pieces/os/ops_registry/<project>.txt
- pieces/os/compiled_projects.txt
- pieces/os/ops_catalog.txt

Moves:
- projects/<project>/ ↔ projects/trunk/<project>/
