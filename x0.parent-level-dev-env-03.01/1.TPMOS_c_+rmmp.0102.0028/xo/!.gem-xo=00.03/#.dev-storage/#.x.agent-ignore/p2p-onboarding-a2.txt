================================================================================
WHAT IS P2P-NET?
================================================================================

P2P-NET is a peer-to-peer networking and messaging system built on TPMOS (True
Piece Method Operating System). It provides:

- Decentralized peer discovery (UDP broadcast)
- Direct peer connections (TCP)
- Email-style messaging between wallets
- Blockchain-based economy (mining, wallets, transactions)
- NFT items, auctions, trading, rentals (future phases)

Think of it as a game-internal P2P network where players can:
- Discover and connect to each other
- Send messages across the network
- Mine coins and trade items
- Create their own economies

All state is file-based. No databases. No shared memory. Files are the database.

================================================================================
ARCHITECTURE OVERVIEW
================================================================================

TPMOS follows a strict layered model:

```
USER INPUT (keyboard)
    ↓
pieces/keyboard/history.txt
    ↓
chtpm_parser.+x (routes to active project)
    ↓
pieces/apps/player_app/history.txt
    ↓
p2p_manager.+x (polls every 16ms)
    ↓
fork/exec → ops/+x/<op_name>.+x
    ↓
Op updates state files (pieces/chat/, pieces/network/, etc.)
    ↓
Manager writes response.txt + state.txt + gui_state.txt
    ↓
Parser substitutes ${variables} in layout
    ↓
current_frame.txt → Display to user
```

Key principle: **Files are the database.** Every op reads/writes files. No memory
sharing. No databases. If it's not in a file, it doesn't exist.

OP IMPLEMENTATION PATTERN
================================================================================

Every op follows this template:

```c
/*
 * op_name.c - Brief description
 * Purpose: What this op does
 * Args: arg1, arg2, ...
 */

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <time.h>
#include <limits.h>

#define MAX_PATH PATH_MAX
#define MAX_LINE 1024

/* Helper: Get base path */
static void get_base_path(char *buf, size_t size) {
    if (getcwd(buf, size) != NULL) return;
    strncpy(buf, ".", size - 1);
}

/* Helper: Ensure directory exists */
static void ensure_dir(const char *path) {
    char cmd[MAX_PATH * 2];
    snprintf(cmd, sizeof(cmd), "mkdir -p %s", path);
    system(cmd);
}

int main(int argc, char *argv[]) {
    char base_path[MAX_PATH];
    
    /* 1. Parse and validate arguments */
    if (argc < EXPECTED_ARGS) {
        fprintf(stderr, "Usage: op_name <arg1> <arg2> ...\n");
        return 1;
    }
    
    /* 2. Get base path */
    get_base_path(base_path, sizeof(base_path));
    
    /* 3. Create directories if needed */
    ensure_dir(some_path);
    
    /* 4. Perform operation (read/write state files) */
    FILE *f = fopen(path, "r");
    // ... do work ...
    
    /* 5. Update state files */
    f = fopen(state_path, "w");
    fprintf(f, "key=value\n");
    fclose(f);
    
    /* 6. Output result */
    printf("Operation completed successfully\n");
    
    /* 7. Return 0 on success, 1 on failure */
    return 0;
}
```

================================================================================
MANAGER PATTERN
================================================================================

The manager is a long-running daemon that follows the CPU-SAFE pattern.

REFERENCE TEMPLATE: #.docs/cpu_safe_module_template.c

Key requirements (from template):
✓ Signal handlers for SIGINT/SIGTERM
✓ setpgid(0, 0) for process group isolation
✓ fork/exec/waitpid (NEVER system() for commands)
✓ stat() before reading files (avoid unnecessary I/O)
✓ is_active_layout() check - sleep 100ms when not in focus
✓ 16ms polling interval (60 FPS max)
✓ Bounded sleep intervals (no tight loops)

Architecture:
```c
int main() {
    /* CPU-SAFE: Setup */
    signal(SIGINT, handle_sigint);
    signal(SIGTERM, handle_sigint);
    setpgid(0, 0);
    resolve_paths();  /* Read project_root from location_kvp */
    
    /* Main loop */
    while (!g_shutdown) {
        /* CPU-SAFE: Focus-aware throttling */
        if (!is_active_layout()) {
            usleep(100000);  /* 100ms when not in focus */
            continue;
        }
        
        /* Poll history file with stat()-first */
        if (stat(history_path, &st) == 0 && st.st_size > last_pos) {
            FILE *f = fopen(history_path, "r");
            fseek(f, last_pos, SEEK_SET);
            
            int key;
            while (fscanf(f, "%d", &key) == 1) {
                process_key(key);  /* Dispatch to op */
            }
            
            last_pos = ftell(f);
            fclose(f);
        }
        
        usleep(16667);  /* ~16ms polling */
    }
    
    return 0;
}

static void process_key(int key) {
    if (key == '1') {
        /* CPU-SAFE: fork/exec/waitpid pattern */
        pid_t pid = fork();
        if (pid == 0) {
            freopen("/dev/null", "w", stdout);
            freopen("/dev/null", "w", stderr);
            execl("/bin/sh", "/bin/sh", "-c", cmd, NULL);
            _exit(127);
        } else {
            waitpid(pid, &status, 0);
        }
        
        /* Update state */
        set_response("Operation completed");
        write_state();
        write_gui_state();  /* For GUI variable substitution */
    }
}
```

CRITICAL: When enhancing p2p_manager.c, follow the cpu_safe_module_template.c
pattern exactly. Do NOT use system(), do NOT skip is_active_layout() checks,
do NOT remove signal handlers.

GUI INTEGRATION PATTERN
================================================================================

The layout (p2p-net.chtpm) expects these variables:

  ${wallet_address}    - Current wallet address
  ${subnet_list}       - Comma-separated subnet names
  ${unread_count}      - Number of unread messages
  ${total_count}       - Total messages in inbox
  ${inbox_list}        - Formatted inbox (first 10 messages)
  ${message_preview}   - Current message content
  ${peer_count}        - Number of connected peers
  ${peer_list}         - Formatted peer list
  ${p2p_response}      - Response from last op
  ${last_key}          - Last key pressed

The manager must write these to `manager/gui_state.txt`:

```
wallet_address=abc123def456...
subnet_list=general, chat, auction
unread_count=1
total_count=1
inbox_list=[!] def456 - Re: Hello!
message_preview=Hi back! Message received...
peer_count=3
peer_list=127.0.0.1:8001 (abc123) [chat] - active
p2p_response=Message sent successfully
last_key=2
```

The parser reads this file and substitutes ${variables} in the layout.

================================================================================
CRITICAL PITFALLS (P2P-NET Specific)
================================================================================

1. COMPILE SCRIPT PATH MISMATCH
   compile_all.sh looks for ops/*.c but sources are in ops/src/*.c
   FIX: Compile manually or update compile_all.sh

2. GETCWD() DEPENDENCY
   All ops use getcwd() to resolve paths
   FIX: Must run from TPMOS root directory

3. NO LOCKING ON SHARED FILES
   Multiple ops can read/write peers.txt concurrently
   FIX: For now, accept race conditions. Add locking if needed later.

4. HARDCODED VALUES IN MANAGER
   Wallet address, recipient, message ID are hardcoded
   FIX: Implement wallet_login, read from session state

5. MANAGER DOESN'T POPULATE GUI VARIABLES
   Only writes response.txt, not gui_state.txt
   FIX: Add write_gui_state() function (see deliverable)

6. P2P-NET IS A PROJECT (Not an APP)
   P2P-NET lives in projects/p2p-net/ (user content, installable via Fondu)
   NOT in pieces/apps/ (system tools)
   Launched via LOAD_PROJECT:p2p-net from loader menu
   REFERENCE: #.docs/op-setup.txt for APP vs PROJECT patterns

7. OPS MUST BE REGISTERED IN ops_manifest.txt
   Format: op_name|description|args_format
   Fondu reads this during installation to register ops system-wide
   REFERENCE: #.docs/02-REFERENCE/OPS_DICTIONARY.txt

8. LEGAL REQUIREMENTS (MUST IMPLEMENT BEFORE LAUNCH)
   - p2p::refund_coins op MUST exist (refund guarantee)
   - p2p::recover_wallet op MUST exist (wallet recovery)
   - All transactions logged for support visibility
   - "Casual Fun" security level (SHA256, not bcrypt/argon2)
   - Support must be able to refund any user, anytime
   - REFERENCE: #.docs/P2P_LEGAL_NOTICE.txt (READ THIS!)

9. SYSTEM-WIDE PITFALLS (from PITFALLS_ACTIVE file)
   #10  - CPU safety (fork/exec, no runaway loops)
   #13  - Backup before editing (no-git mode)
   #18  - State/PDL integrity (section-aware updates)
   #20  - Resolve project_root from location_kvp
   #25  - Persist responses to files, not stdout
   #46  - Compile all binaries before testing

   #70  - Report actual outcomes, not assumed success
