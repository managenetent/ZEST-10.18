ARCHITECTURE OVERVIEW
================================================================================

TPMOS has TWO types of executable entities:

1. APPs (pieces/apps/<app_id>/)
   - System infrastructure and utilities
   - Launched via LAUNCH:<APP_NAME> command or button href
   - Module runs in background, writes to pieces/apps/<app>/state.txt
   - Examples: player_app (input buffer), chtpm (parser), keyboard, display

2. PROJECTs (projects/<project_id>/)
   - User content, games, and installable apps
   - Launched via Loader Menu or Fondu installation
   - Module runs, writes to projects/<project>/manager/state.txt
   - Can be installed to pieces/apps/installed/ via Fondu
   - Examples: op-ed, fuzz-op, user, test_fondu, gl-os

NOTE: Projects can be deployed as production apps via Fondu.
      See: #.docs/fondu-investigation-report.txt

================================================================================
LAUNCH MECHANISMS
================================================================================

1. LAUNCH:<APP_NAME> (for APPs)
   - Reads from pieces/os/app_list.txt
   - Format: APP_NAME=path/to/module.+x
   - Example: GL-OS=pieces/apps/gl_os/plugins/+x/gl_desktop.+x
   - Does NOT set project_id
   - Module runs in background

2. LOAD_PROJECT:<PROJECT_ID> (for PROJECTs)
   - Reads from pieces/os/project_routes.kvp
   - Format: project_id=path/to/layout.chtpm
   - Sets project_id in pieces/apps/player_app/manager/state.txt
   - Sets active_target_id=xlector (for most projects)
   - Module runs, manages project state

================================================================================
STATE FILE LOCATIONS
================================================================================

APPs:
- pieces/apps/<app>/state.txt (module state)
- pieces/apps/<app>/session/state.txt (session state, e.g., GL-OS)

PROJECTs:
- projects/<project>/manager/state.txt (project state)
- pieces/apps/player_app/manager/state.txt (shared engine state)

SHARED:
- pieces/apps/player_app/history.txt (input buffer)
- pieces/display/current_layout.txt (current layout path)
- pieces/os/compiled_projects.txt (Fondu manifest)
- pieces/os/ops_registry/<project>.txt (Fondu ops registry)

================================================================================
COMMON ISSUES AND FIXES
================================================================================

1. GL-OS shows wrong layout (op-ed map instead of desktop)
   CAUSE: project_id was set, parser loads wrong state
   FIX: Remove set_var("project_id", "gl-os") from parser
   FIX: Ensure gl_desktop.c writes ALL state variables

2. op-ed xlector moves randomly on launch
   CAUSE: Reading old keys from history buffer
   FIX: Add stat() check before main loop to skip existing keys
   CODE: if (stat(hist_path, &st) == 0) last_pos = st.st_size;

3. Project not found in loader
   CAUSE: Missing from project_routes.kvp
   FIX: Add entry: project_id=path/to/layout.chtpm

4. Ops not working
   CAUSE: Binary not compiled or symlink broken
   FIX: Compile ops, check symlinks point to correct location

================================================================================
APP LAUNCH PATTERNS
================================================================================

LAUNCH:<APP_NAME> (for system apps)
─────────────────────────────────────────────────────────
Source: pieces/os/app_list.txt
Format: APP_NAME=path/to/module.+x
Examples:
  GL-OS=pieces/apps/gl_os/plugins/+x/gl_desktop.+x
  RMMP=pieces/apps/editor/plugins/+x/editor_module.+x
  PLAYR=pieces/apps/playrm/plugins/+x/playrm_module.+x

Does NOT set project_id
Module runs in background (fork + exec)
Layout determined by module

LOAD_PROJECT:<PROJECT_ID> (for user projects)
─────────────────────────────────────────────────────────
Source: pieces/os/project_routes.kvp
Format: project_id=path/to/layout.chtpm
Examples:
  op-ed=projects/op-ed/layouts/op-ed.chtpm
  fuzz-op=pieces/apps/playrm/layouts/game.chtpm
  user=pieces/apps/user/layouts/user.chtpm

Sets project_id in pieces/apps/player_app/manager/state.txt
Sets active_target_id=xlector (for most projects)
Layout determined by project_routes.kvp

================================================================================
STATE FILE LOCATIONS
================================================================================

APPs (pieces/apps/<app_id>/):
  - pieces/apps/<app>/state.txt (module state)
  - pieces/apps/<app>/session/state.txt (session state, e.g., GL-OS)
  - pieces/apps/<app>/session/view.txt (view content, e.g., GL-OS desktop)

PROJECTs (projects/<project_id>/):
  - projects/<project>/manager/state.txt (project state)
  - projects/<project>/pieces/<piece>/state.txt (piece state)

SHARED:
  - pieces/apps/player_app/history.txt (input buffer)
  - pieces/apps/player_app/manager/state.txt (engine state)
  - pieces/display/current_layout.txt (current layout path)
  - pieces/os/app_list.txt (APP launch routes)
  - pieces/os/project_routes.kvp (PROJECT launch routes)
