## PROBLEM SUMMARY

GL-OS terminal menu navigation has MULTIPLE bugs:

1. ❌ **Menu only shows 1 item** - "2. [Terminal Apps]" is MISSING
2. ❌ **Duplicate "1." indexes** - Window list has "1." AND menu has "1." (breaks ASCII-OS jump navigation)
3. ❌ **Joystick not updating selection** - Selection changes but frame doesn't show item 2

---

## ROOT CAUSE ANALYSIS

### Bug 1: Multi-line State File Variables Don't Work

**Problem:** The parser reads state file line-by-line:
```c
while (fgets(line, sizeof(line), state)) {
    char *eq = strchr(line, '=');
    // ...
}
```

But `menu_options` has newlines:
```
menu_options=  [>] 1. [Desktop Apps]
  [ ] 2. [Terminal Apps]
```

**Result:** Only the first line is captured. Second line is lost.

**ASCII-OS Reference:** ASCII-OS does NOT use multi-line state variables. It builds the frame DIRECTLY from parsed layout elements using `render_element()`.

---

### Bug 2: Duplicate Index Numbers

**Current GL-OS:**
```
║    [>] 1. Terminal 1          ← Window list uses "1."
║  MENU (Joystick UP/DOWN to navigate)
║    [>] 1. [Desktop Apps]      ← Menu ALSO uses "1."
```

**Problem:** User can't use "jump" navigation (typing "2" to select second item) because both sections use the same numbers.

**ASCII-OS Reference:** ASCII-OS has ONE unified navigation list. All navigable items share a single index sequence.

---

## SOLUTION

### Fix 1: Build Frame Directly (Like ASCII-OS)

**Remove the broken state-file-parser approach.** Instead:
1. Build frame DIRECTLY in `write_gl_os_state()`
2. Write complete frame to `current_frame.txt`
3. Don't use multi-line state variables

**OR**

### Fix 2: Encode Multi-line as Single Line

Use a delimiter (e.g., `|`) to encode newlines:
```
menu_options=  [>] 1. [Desktop Apps]|[ ] 2. [Terminal Apps]
```

Then decode in parser. But this is hacky.

---

### Fix 3: Separate Index Spaces

Window list and menu should have DIFFERENT index schemes:
```
Window list: Use letters (A, B, C) or no numbers
Menu: Use numbers (1, 2, 3)
```

OR remove window list entirely and just show menu.

## ROOT CAUSE ANALYSIS

### Bug 1: Extra `║` Character in Menu Lines

**Location:** `gl_desktop.c` line ~355

**Current Code:**
```c
snprintf(line, sizeof(line), "║  %s %d. [%s]                            ║\n", 
         indicator, i + 1, menu_items[i]);
```

**Problem:** The format string includes `║` at the START of each line, but these lines are already INSIDE a bordered frame. This creates double borders.

**Fix:**
```c
snprintf(line, sizeof(line), "  %s %d. [%s]\n", 
         indicator, i + 1, menu_items[i]);
```

---

### Bug 2: Window List Also Has Extra `║`

**Location:** `gl_desktop.c` line ~323

**Current Code:**
```c
snprintf(line, sizeof(line), "║  %s %d. %-20s            ║\n",
         indicator, i + 1, windows[i].title);
```

**Problem:** Same issue - extra `║` at start.

**Fix:**
```c
snprintf(line, sizeof(line), "  %s %d. %-20s\n",
         indicator, i + 1, windows[i].title);
```

---

### Bug 3: Frame Not Being Written on Every Selection Change

**Location:** `gl_desktop.c` joystick() function, line ~1155

**Current Code:**
```c
/* Write new frame if selection changed */
if (selection_changed) {
    printf("[JOY NAV] Selection changed, writing new frame...\n");
    write_gl_os_state();
    parse_and_write_frame();
}
```

**Problem:** The `selection_changed` flag is set, but the frame history shows the SAME frame repeated (same timestamp 08:02:04). This means either:
1. `parse_and_write_frame()` is not being called
2. `write_gl_os_state()` is not updating the state correctly
3. The frame is being written but with WRONG state

**Evidence:** Frame history shows identical frames:
```
--- FRAME UPDATE at 2026-03-24 08:02:04 ---
--- FRAME UPDATE at 2026-03-24 08:02:04 ---  ← Same timestamp!
```

**Fix:** Add debug output to verify the full cycle:
```c
if (selection_changed) {
    printf("[JOY NAV] Selection changed, writing new frame...\n");
    printf("[JOY NAV] selected_idx: %d\n", windows[i].selected_index);
    write_gl_os_state();
    printf("[JOY NAV] State written\n");
    parse_and_write_frame();
    printf("[JOY NAV] Frame written\n");
}
```

---

### Bug 4: `selected_index` Not Persisted in State File

**Location:** `gl_desktop.c` write_gl_os_state() function

**Problem:** The `selected_index` is used to build the menu, but it's not written to the state file for the parser to use. The state file should contain:
```
selected_index=1
```

**Fix:** Add selected_index to state file:
```c
void write_gl_os_state(void) {
    /* ... existing code ... */
    
    /* Get selected index */
    int selected_idx = 0;
    for (int i = 0; i < window_count; i++) {
        if (windows[i].id == active_window_id && 
            strcmp(windows[i].type, "terminal") == 0) {
            selected_idx = windows[i].selected_index;
            break;
        }
    }
    
    /* Write selected_index to state file */
    fprintf(f, "selected_index=%d\n", selected_idx);
    
    /* ... rest of code ... */
}
```
