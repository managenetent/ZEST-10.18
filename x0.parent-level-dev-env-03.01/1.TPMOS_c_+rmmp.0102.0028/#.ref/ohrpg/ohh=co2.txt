NOTE: This is the DEFAULT menu structure. Game authors can create custom menus
by editing menus in the Custom editor or through script triggers. The tree below
shows typical relationships and hierarchies.

================================================================================
                          MENU TREE STRUCTURE LEGEND
================================================================================

Menu Notation:
  [ID] Menu Name
    Type of menu (Special/Standard/Conditional)
    
Item Links:
  → Links to another menu (submenu)
  ⊕ Opens dialog/confirmation
  ♦ Runs a script/trigger
  ★ Special built-in behavior
  ◊ Conditional (appears if tag/condition met)

Prefixes:
  • Auto-close item
  ○ Non-closeable item
  ◌ Hidden when disabled
  ✓ Selectable by default

================================================================================
                    MENU CLOSING & ESC BEHAVIOR
================================================================================

ESC KEY BEHAVIOR
  [0] Main Menu
    └─ ESC → Close menu, resume game
  
  [1-5] Settings Submenus
    └─ ESC → Return to parent menu
  
  [10-13] Shop Menus
    └─ ESC → Close shop, resume game
  
  [20-22] Battle Menus
    └─ ESC → Return to previous menu (no exit from battle)
  
  [30+] Event Menus
    └─ ESC → Depends on "no_close" flag
           If NO_CLOSE: Cannot close with ESC
           Otherwise: Close menu, resume game

ON_CLOSE SCRIPT CALLBACKS
  After menu closes (if set):
    [12] Sell Menu → Script: Process item sale
    [11] Buy Menu → Script: Process item purchase
    [13] Hire Menu → Script: Update party roster
    [20+] Battle Menus → Scripts: Not typically used
    [30+] Event Menus → Scripts: Update quest flags

================================================================================
                        MENU ACCESS FROM SCRIPTS
================================================================================

SCRIPT COMMANDS FOR MENU CONTROL
  open_menu(record_id)
    Opens menu by ID
    Pushes to menu stack
    Returns to menu after script completes
  
  close_menu()
    Closes top menu
    Executes on_close script
    Returns to previous menu
  
  get_menu_selection()
    Returns currently selected item index
    Useful in menu callbacks
  
  set_menu_selection(index)
    Changes selected item
    Updates display on next frame
  
  menu_is_open(record_id)
    Returns YES if menu is in stack
    Useful for conditional logic

MENU ITEM TYPES FOR SCRIPTS
  mtypeScript
    Item runs specified plotscript
    Passes item data as arguments
    Menu stays open unless close_when_activated set
  
  mtypeMenu
    Item opens another menu
    New menu pushes to stack
    Previous menu resumes when new closes
  
  mtypeTextBox
    Item displays textbox
    Returns selection/input from textbox
    Menu remains active behind textbox

================================================================================
                    CUSTOM MENU DEVELOPMENT GUIDE
================================================================================

CREATING A CUSTOM MENU
  1. Assign new record ID (e.g., 50)
  2. Set menu properties:
     - Name, box style, colors
     - Positioning (anchor, offset)
     - Behavior flags (translucent, allow_gameplay, etc.)
  3. Add menu items:
     - Caption and type
     - Navigation or script references
     - Conditional tags
  4. Set callbacks:
     - on_close script for post-menu processing
     - esc_menu for ESC behavior
  5. Open from game via:
     - Script: open_menu(50)
     - Menu item: mtypeMenu linking to ID 50
     - NPC activation: triggered by event

MENU NESTING LIMITS
  - Maximum stack depth: Limited by available memory
  - Typical depth: 2-5 menus (start menu → submenu → item selector)
  - Safe practice: Don't exceed 10 nested menus
  - ESC key always closes top menu, walks stack backwards

PERFORMANCE CONSIDERATIONS
  - Menus are relatively lightweight
  - String drawing can be expensive with many items
  - Sorting large item lists may cause frame hitches
  - Use "no_controls" flag if menu is display-only
  - Consider pagination for 50+ item lists

================================================================================
                         SPECIAL MENU SITUATIONS
================================================================================

NO-CLOSE MENUS
  Used for: Dialog boxes, tutorials, locked menus
  Behavior: ESC doesn't work, must activate item
  Example:
    [99] Important Message
      ├─ "Click to Continue"
      │  └─ • Close menu
      └─ (no other options, no_close=YES)

MENU CHAINS
  Used for: Purchasing flow, multi-step dialogs
  Behavior: Each item opens next menu via esc_menu
  Example:
    [50] Item Selected
      ├─ "How many? → [51] Quantity Menu
      └─ "Cancel" ESC closes
    
    [51] Quantity Menu
      ├─ Slider for quantity
      └─ "Buy" → [52] Confirm Menu
    
    [52] Confirm Purchase
      ├─ "Yes - Complete purchase" (script: deduct gold)
      └─ "No - Cancel" ESC back to [51]

CONDITIONAL MENUS
  Used for: Dynamic options based on game state
  Behavior: Items show/hide based on tags or script values
  Example:
    [60] Adventure Options
      ├─ "Enter Cave" ◌ (tag 10=quest active, hidden if tag off)
      ├─ "Visit Inn" (always available)
      ├─ "Speak to Mayor" ◌ (tag 5=haven't talked, hidden if already done)
      └─ "Leave Town"

FLOATING MENUS (HUDs)
  Used for: Status displays, notifications
  Behavior: Drawn over game, no interaction needed
  Example:
    [99] Game HUD
      ├─ "HP: {hp}/{max_hp}"
      ├─ "MP: {mp}/{max_mp}"
      └─ "Funds: {gold}" (gold)
      Properties: translucent=YES, no_box=YES, no_controls=YES

================================================================================
                          COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE: Menu won't close
  CAUSE: no_close=YES or esc_menu set to loop back
  FIX: Add "Cancel" item with close_when_activated=YES
  OR: Script calls close_menu() explicitly

ISSUE: Selected item won't activate
  CAUSE: Item is disabled (disabled=YES or tag condition failed)
  FIX: Check tag values, enable item in editor
  OR: Menu has no_controls=YES (menu is display-only)

ISSUE: Menu appears off-screen
  CAUSE: Positioning outside screen bounds
  FIX: Check anchorhoriz/anchorvert/offset values
  OR: Enable clamp_to_screen flag

ISSUE: Menu items don't show
  CAUSE: hide_if_disabled=YES with all items disabled
  OR: itemlist is empty (check LoadMenuData)
  FIX: Disable tag conditions, add visible items

ISSUE: Menu too slow / framerate drop
  CAUSE: Drawing too many items each frame
  OR: On_close script is slow
  FIX: Pagination, use no_controls for display-only
  OR: Optimize script execution

ISSUE: Previous menu still visible under new menu
  CAUSE: New menu has translucent=YES or no_box=YES
  FIX: Add opaque background, check visual design
  (This is sometimes intentional for HUD-style menus)
