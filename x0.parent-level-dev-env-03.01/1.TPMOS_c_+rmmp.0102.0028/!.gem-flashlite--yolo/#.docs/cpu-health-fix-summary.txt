## How It Works

1. **On startup:** Orchestrator clears `proc_list.txt`, registers itself
2. **On spawn:** Each child PID is logged to `proc_list.txt` with flock
3. **On Ctrl+C:** Signal handler reads `proc_list.txt`, kills all PIDs (SIGTERM→200ms→SIGKILL), clears file
4. **Process groups:** Each child is its own group leader (`setpgid(0,0)`), so `kill(-PID, SIG)` kills the child AND any grandchildren
