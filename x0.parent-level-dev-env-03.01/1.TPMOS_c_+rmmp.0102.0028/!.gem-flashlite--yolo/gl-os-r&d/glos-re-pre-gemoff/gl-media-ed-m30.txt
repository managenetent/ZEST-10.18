GL-OS Media Editor (media-editor) - Design & Integration Blueprint
===============================================================
Date: May 30, 2026
Version: 1.0 (m30)

1. PROJECT OVERVIEW
-------------------
The `media-editor` is a rich media authoring tool for GL-OS. It will support dragging and dropping assets, scene manipulation, and animation scripting via PRISC.

2. CHTMGL FEATURES TO INTEGRATE
-------------------------------
To support a professional media editor, the following CHTMGL features must be ported to TPMOS (starting with `chtmgl-alpha`):

- **Advanced <canvas> (3D/2D):**
  - Need to support `.obj` and `.fbx` (animation) loading directly into a canvas element.
  - Implement "Gizmos" (translation/rotation/scale handles) as part of the canvas interaction layer.
  - Add `view_mode="editor"` to enable grid lines and axes.

- **Media Tags with Metadata:**
  - `<img>`, `<audio>`, `<video>` need attributes for `autoplay`, `loop`, and `volume`.
  - Integration with `stb_image` and `ffmpeg` (via tasks/ops) for thumbnail generation and stream scrubbing.

- **Property Inspector Elements:**
  - `<slider>` and `<textfield>` for real-time attribute editing (e.g., scale, position, color).
  - `<checkbox>` for toggling visibility or physics properties.

- **Timeline UI:**
  - A specialized `<scroller>` or custom `<timeline>` tag for keyframe management.

3. INTEGRATION INTO CHTMGL-ALPHA
--------------------------------
- **Phase 1 (Assets):** Implement a file-picker/drop-zone that updates `gui_state.txt` with absolute paths.
- **Phase 2 (3D Renderer):** Update `gl_renderer.c` to handle 3D mesh rendering within element bounds.
- **Phase 3 (IPC):** Use Mode 1 (Direct C) for gizmo manipulation to ensure sub-16ms latency. Use Mode 2 (PRISC) for multi-object animation sequences.

5. MANAGER LOGIC (manager/editor_manager.c)
------------------------------------------
- **State Management:**
  - `selected_piece_id`: Tracks which item in the canvas is active.
  - `asset_cache`: A list of discovered media files in the project's `assets/` folder.
- **IPC Handlers:**
  - `SLIDER:time_scrub:<val>` -> Updates the scene state at time <val>.
  - `CANVAS:editor_viewport:<x>,<y>` -> Raycasts to find clicked object, updates `selected_piece_id`.
  - `EVENT:apply_props` -> Writes new coordinates to the Piece's `state.txt` and hits `frame_changed.txt`.
- **PRISC Integration:**
  - Animations are stored as `.asm` files in `scripts/animations/`.
  - The manager calls `prisc+x <script_path>` to execute frame-by-frame updates during playback.
