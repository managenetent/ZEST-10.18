# LSR Project Handover Report: Path to Self-Evolving Simulation

## 1. User Goals & Vision

This report details the current state and future roadmap for the LSR (Lunar Streetrace Raider) project, with a focus on creating a self-improving, intelligent simulation. The primary objectives are:

👉 **Augment the existing plan with a macro Phase 2 (Learning + Agents + Curriculum)**
👉 **Push the map / z-layer vision into a clean Phase 3**

The existing plan structure will be maintained and layered upon to ensure seamless integration and avoid breaking current progress.

# 🧠 2. What You Already Did Right (Current LSR State)

The current LSR project, built upon the WSR foundation, has successfully established:

*   **System First Approach:**
    *   **Phase 1:** Economy Ops (Metabolism) – Core financial loops are being integrated as standalone Ops.
    *   **Phase 2:** Piece Migration – Data is being converted into TPM Pieces (`state.txt` and `.pdl`).
    *   **Phase 3:** Manager (Exo Agent) – A basic manager bot is being implemented.
    *   **Phase 4:** Geography (Map Container) – Initial structure for map containers is being laid out.
    *   **Phase 5:** AI R&D – The foundation for AI-driven research is planned.

This adheres to the correct principle of building the foundational systems before introducing advanced agent capabilities.

This gap is the focus of the new macro Phase 2 addition.

# 🧩 4. AUGMENTED PLAN (Clean Integration)

# 🚀 🟪 PHASE 2X: LEARNING + CURRICULUM ENGINE (NEW)

*(This phase runs AFTER the current Phase 1–3 stability is achieved.)*

## 🎯 Goal
Turn LSR from a simulation into a **self-improving system**.

## 🟥 2X.1 — Curriculum System
Create a structured learning environment:
```plaintext
lsr/pieces/curriculum/
```
Each curriculum piece will define:
```plaintext
fire_v1/
  goal.txt            # e.g., "increase profit"
  environment.txt     # context for the learning agent
  actions.txt         # possible actions the agent can take
  reward.txt          # function to calculate agent's reward
  success.txt         # criteria for successful completion
```
**Example (business version):**
```plaintext
profit_growth_v1/
  goal: increase profit
  actions: hire, fire, invest, price_change
  reward: revenue - cost
  success: profit > baseline
```

## 🟧 2X.2 — Agent Training Loop
Extend the EXO bot FSM to incorporate learning:
**Current FSM:**
```plaintext
SCAN → CALCULATE → EXECUTE → LEARN
```
**Upgraded FSM:**
```plaintext
SCAN
→ SELECT CURRICULUM
→ EXECUTE ACTIONS
→ EVALUATE (CRITICAL)
→ UPDATE WEIGHTS
→ STORE SKILL
```
This loop is central to the agent's ability to adapt and improve over time.

## 🟨 2X.3 — Evaluator System (MANDATORY)
Create a dedicated system for evaluating agent actions and performance:
```plaintext
xo/lsr_manager/evaluator/
```
Outputs will include:
```plaintext
evaluation.txt
score.txt
reason.txt
```
This directly connects to the M18 testing system, where `frame diff + state diff = ground truth reward`.

## 🟩 2X.4 — Knowledge Distillation
Establish a system for agents to share and reuse learned knowledge:
```plaintext
lsr/pieces/knowledge/
```
Example structure:
```plaintext
pricing_strategy_v1/
  inputs.txt
  actions.txt
  outcomes.txt
  confidence.txt
```
**Flow:** Successful runs → extract pattern → store → reuse.

## 🟦 2X.5 — CLI AI Integration (Gemini CLI)
Leverage Google Gemini CLI for advanced AI tasks:
*   Knowledge generation
*   Evaluation assistance
*   Distillation processes

An example operation could be:
```plaintext
lsr/ops/knowledge_extract_op.+x
```

# 🌍 🟫 PHASE 3X: MAP + Z-LAYER SYSTEM (CLEANED VISION)

This phase refines the spatial and hierarchical structure for the simulation world.

## 🎯 Core Idea
```plaintext
everything exists on a map
+ each entity has its own z-layer
```

## 🟥 3X.1 — Base Map (LSR World Layer)
Define the global map container:
```plaintext
lsr/pieces/world/lunar/map_alpha/
```
This layer will contain:
*   Corporations
*   Agents
*   Landmarks

## 🟧 3X.2 — Entity Z-Layers
Each entity will have its own hierarchical Z-layers for managing complexity:
```plaintext
corp_001/
  z_layers/
    z0_world_link.txt     # Exists on global map
    z1_internal/          # Internal operations
    z2_detail/            # Deep simulation (factory, lab, etc.)
```
This structure allows for seamless zooming from macro (world view) to micro (entity details).

## 🟨 3X.3 — Selector Teleport (Key Interaction)
Enhance interaction by allowing seamless navigation:
When selecting an entity:
```plaintext
selector_piece → teleport → entity.z_layer
```
And in reverse:
From inside an entity:
```plaintext
exit → return to world map
```
This provides a **seamless zoom (macro ↔ micro)** experience.

## 🟩 3X.4 — LSR Interface Hook (Entity Methods)
Standardize entity interfaces for interaction:
Implement a standard `lsr` method for all entities:
```plaintext
METHOD | lsr_enter | void
```
This allows direct entry into an entity's Z-layer UI when selected:
Flow:
```plaintext
select entity → call lsr_enter → load z-layer UI
```

## 🟦 3X.5 — Menu vs Map Structure
The menu system will function as a **navigation layer**, while the map-first approach provides a more immersive and scalable experience.
