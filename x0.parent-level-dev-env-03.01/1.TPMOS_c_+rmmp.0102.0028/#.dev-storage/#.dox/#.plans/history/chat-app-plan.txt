# Handoff Plan: Chat Application Development in TPMOS

## 1. Objective
Develop a functional chat application within the TPMOS ecosystem, leveraging existing P2P operations for real-time communication. The application should allow users to send and receive messages.

## 2. Environment Overview
*   **System:** TPMOS CLI (v2.0), running on Linux.
*   **Project Structure:** Standard TPMOS layout under `projects/<project_id>/`. Key directories include `manager/` for core logic, `ops/` for external operations, `layouts/` for UI definitions (`.chtpm`), and `project.pdl` for project metadata.
*   **Tools & Conventions:**
    *   C language for manager and ops.
    *   `.chtpm` for UI layouts.
    *   `project.pdl` for project metadata.
    *   `button.sh` and `compile_all.sh` for build processes (though `chat-app` is currently excluded from compilation).
    *   Manual registration steps are in place for projects and ops.
    *   P2P operations are fundamental to communication.

## 3. Current Status
*   **Project Setup:** Directories `projects/chat-app/`, `layouts/`, `manager/`, `ops/`, `ops/src/` are created.
*   **Project Descriptor:** `projects/chat-app/project.pdl` is populated with basic metadata.
*   **Layout:** `projects/chat-app/layouts/chat-app.chtpm` is defined with a message display area and an input field, but lacks dynamic content population.
*   **Registration:** `chat-app` is registered as a compiled project and its ops (`send_message`, `register_user`, `get_messages`) are listed in `ops_registry/chat-app.txt` and `ops_catalog.txt`.
*   **Manager/Ops Code:** C source files for the manager and ops were created but failed to compile due to C syntax errors (string literals, argument parsing) and potentially missing TPMOS build context. These files have been deleted to avoid further compilation issues for now.
*   **Build System:** `chat-app` is intentionally excluded from compilation via `compile_all.sh` modifications (or by absence of compilable code).

## 4. Handoff Agent's Role
Your task is to take over development for the `chat-app` project. You will focus on resolving the compilation issues for the manager and ops, implementing core chat functionality, and ensuring the layout is interactive.

## 5. Stepwise Development Plan

### Phase 1: Resolve Compilation and Basic Manager Setup
*   **Objective:** Get a compilable C manager and essential ops.
*   **Steps:**
    1.  **Investigate C Compilation Errors:** Analyze the previous compilation failures. Focus on understanding the correct C syntax for string literals (especially with newlines), argument parsing, and any TPMOS-specific compiler flags or includes required for managers and ops.
    2.  **Create Corrected Manager C Code:** Write a basic `chat-app_manager.c` that includes essential TPMOS patterns (signal handling, basic input polling placeholder, op calling placeholder, render trigger placeholder) with correct C syntax. Ensure it can compile successfully with `gcc` or `button.sh`.
    3.  **Create Corrected Op C Code:** Write minimal, compilable C stubs for `send_message`, `register_user`, and `get_messages` ops. These should compile successfully.
    4.  **Compile and Verify:** Attempt to compile the manager and ops. Ensure they produce valid binaries.

### Phase 2: Basic Chat Functionality (Manager & Ops Interaction)
*   **Objective:** Enable sending and receiving of simple text messages.
*   **Steps:**
    1.  **Implement Event Handling in Manager:** Modify `chat-app_manager.c` to listen for the `SUBMIT_EVENT` from the `chat-app.chtpm` layout's input field.
    2.  **Call `send_message` Op:** When the submit event is triggered, the manager should call the `send_message` op with the input text.
    3.  **Implement `get_messages` Op and Manager Retrieval:** Implement the `get_messages` op to simulate fetching messages. Modify the manager to periodically call `get_messages` and receive (even placeholder) message data.
    4.  **Populate Layout Data:** Update the manager to populate the `chat_messages` variable in `chat-app.chtpm` with received messages.

### Phase 3: User Input and P2P Integration
*   **Objective:** Enhance user identification and integrate P2P communication logic.
*   **Steps:**
    1.  **Implement User Registration:** Integrate the `register_user` op in the manager to allow setting a username.
    2.  **Refine `send_message` Op:** Ensure messages sent include sender information.
    3.  **Refine `get_messages` Op:** Implement basic P2P message retrieval logic.
    4.  **Display Messages with User Info:** Update the manager and layout to display messages with sender names and timestamps.

### Phase 4: Refinement, Testing, and Integration
*   **Objective:** Ensure the application is robust and well-integrated.
*   **Steps:**
    1.  **Refine `chat-app.chtpm` Layout:** Ensure the UI is clean, basic, and functional.
    2.  **Testing:** Write and run tests for core functionalities.
    3.  **Final Compilation & Registration:** Re-compile and ensure the project registers correctly.

## 6. Key Deliverables & KPIs
*   **Deliverable:** A functional chat application that can send and receive messages.
*   **KPIs:**
    *   Message Delivery: Messages sent are reliably received within 5 seconds.
    *   UI Responsiveness: Message display updates within 1 second.
    *   User Identification: Users can be uniquely identified.
    *   Project Integration: Project compiles, registers, and loads.

## 7. Additional Guidance
*   When researching user input and interaction, consult existing "user" project functionality or other core system components.
*   Prioritize resolving C compilation issues for the manager and ops.
*   Adhere strictly to TPMOS coding conventions and architectural patterns.

Please begin by tackling Phase 1, Step 1 & 2: Resolving C compilation issues and creating a basic, compilable manager.
