GL-OS FINAL FIX (April 2, 2026)
================================================================================

ISSUE: GL-OS ASCII layout showed empty values for ${bg_color}, ${window_count}, ${desktop_view}

ROOT CAUSE: Parser's load_vars() function wasn't loading GL-OS session state.
- load_vars() loaded: pieces/apps/player_app/state.txt
- load_vars() loaded: pieces/apps/player_app/manager/state.txt
- load_vars() DID NOT load: pieces/apps/gl_os/session/state.txt

The GL-OS specific variables (bg_color, window_count, desktop_view, gl_response)
were written to pieces/apps/gl_os/session/state.txt by gl_desktop.c, but the
parser never loaded them for variable substitution.

FIX APPLIED:
Added GL-OS session state loading in load_vars() function:

    // GL-OS SESSION STATE (loads bg_color, window_count, desktop_view, etc.)
    if (strstr(current_layout, "pieces/apps/gl_os/layouts/desktop.chtpm") != NULL) {
        load_state_file("pieces/apps/gl_os/session/state.txt", NULL);
    }

This ensures that when the GL-OS desktop layout is active, the parser loads
the GL-OS session state file which contains:
- bg_color=#000044
- window_count=N
- desktop_view=DESKTOP_VIEW\nBG: #000044\nWINDOW_COUNT: N\nWINDOW | ...
- gl_response=

STATE FILE LOADING ORDER:
1. pieces/apps/player_app/state.txt (base state)
2. pieces/apps/player_app/manager/state.txt (engine state)
3. pieces/apps/gl_os/session/state.txt (GL-OS specific - loaded LAST to override)
