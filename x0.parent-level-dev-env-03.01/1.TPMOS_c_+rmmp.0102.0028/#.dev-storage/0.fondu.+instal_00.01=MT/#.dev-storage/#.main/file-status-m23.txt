## SAVE AS Implementation Plan

### Overview
Use the new `cli_io` input persistence to let user edit project name, then clone current project directory.

### UI Flow
1. User clicks "SAVE AS" in file_menu.chtpm
2. Switch to `save_as_prompt.chtpm` layout
3. Show `cli_io` field with current project name pre-filled
4. User edits name, presses Enter
5. Module clones project directory with new name
6. Switch to new project context

### Clone Logic
```c
void clone_project(const char* new_name) {
    char src_path[MAX_PATH], dst_path[MAX_PATH];
    
    // Source: projects/<current_project>/
    snprintf(src_path, sizeof(src_path), "%s/projects/%s", 
             project_root, current_project);
    
    // Destination: projects/<new_name>/
    snprintf(dst_path, sizeof(dst_path), "%s/projects/%s", 
             project_root, new_name);
    
    // Create destination directory
    char cmd[MAX_CMD];
    snprintf(cmd, sizeof(cmd), "mkdir -p '%s'", dst_path);
    run_command(cmd);
    
    // Copy all contents recursively
    snprintf(cmd, sizeof(cmd), "cp -r '%s'/* '%s'/", src_path, dst_path);
    run_command(cmd);
    
    // Update project.pdl with new project_id
    // Update state files with new project context
    
    // Switch editor context to new project
    strncpy(current_project, new_name, sizeof(current_project)-1);
    
    // Write response
    write_editor_response("Saved as: %s", new_name);
    
    // Return to editor
    // (layout switch via state trigger)
}
```
