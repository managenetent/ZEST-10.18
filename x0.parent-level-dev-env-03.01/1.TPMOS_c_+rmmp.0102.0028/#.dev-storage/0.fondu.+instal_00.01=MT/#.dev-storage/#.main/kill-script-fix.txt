HOW IT WORKS:
-------------

The surgical_kill() function uses path-specific regex patterns:
  - pieces/.*/\+x/${name}\.\+x
  - projects/.*/\+x/${name}\.\+x
  - pieces/system/.*/${name}

This ensures ONLY our binaries in our directories are killed, not system
processes or browsers that happen to have similar names.

Example: pkill -f "orchestrator" might kill browser tabs with "orchestrator"
         in the title. surgical_kill "orchestrator" only kills:
         pieces/*/plugins/+x/orchestrator.+x

USAGE:
------

./button.sh k         # Kill all CHTPM processes (safe)
./pieces/os/kill_all.sh  # Direct call

WHAT GETS KILLED:
-----------------

System: orchestrator, chtpm_parser, renderer, keyboard_input, etc.
Legacy: fuzzpet_manager, playrm_module, etc.
System Apps: op-ed_module, man-*-module, user_module, etc.
Fondu Projects: test_fondu_manager, user_manager, fuzz-op_manager
Ops: move_*, interact, render_map, project_loader, etc.

NOT killed: browsers, system tools, unrelated processes.
