# Informed AI Plan: Quiz Platform & Cross-App Integration (TPM-AI)

## 1. Project Overview: Quiz-Bot
**Title:** Quiz-Bot (TPM-AI Quiz Platform)
**Path:** `projects/quiz-bot/`
**Objective:** A "Piece-Op" compliant quiz platform that uses manual user data to train AI bots (FSM, RL, Attention) for use in testing, games (LSR), and knowledge distillation.

## 2. Architecture (Piece-Op Format)
The current `mcat-quiz` C-based CLI will be refactored into a TPMOS project:

- **manager/quiz_orchestrator.+x**: Handles the session loop, loading questions from pieces, and evaluating answers.
- **pieces/question_bank/**: A container piece where each question is a sub-piece.
    - `pieces/question_bank/q_001/state.txt`: `name=Aspartame, formula=C14H18N2O5, type=Amino Acid, groups=Amide; Ester, relevance=Artificial sweetener...`
- **pieces/ai_tutor/**: Stores the FSM state and Attention weights for the AI tutor.
- **pieces/user_profile/**: Stores `rl_buffer.txt` (raw data for RL training) and `history.txt`.
- **layouts/quiz.chtpm**: The UI for the quiz, compatible with GL-OS/CHTPM.

## 3. Recursive AI Architecture (FSM vs RL vs Attention)

The AI is implemented as a "Stacked" recursive system, allowing for both deterministic control and learned behavior.

### A. The "Skeletal" FSM (Deterministic Controller)
- **Role:** High-level state management (UI flow, error handling, session lifecycle).
- **PAL Ops:** `bot::set_state`, `bot::get_state`.
- **Logic:** Decides *what* can happen next based on hard constraints.

### B. The "Policy" RL (Strategy Optimizer)
- **Role:** Optimization of stochastic decisions (e.g., question selection difficulty).
- **PAL Ops:** `ai::decide`, `ai::reward`.
- **Logic:** Analyzes `rl_buffer.txt` to select an action that maximizes the "Learning Rate" reward.
- **Recursion:** The RL agent's output is fed back as a "target_state" for the FSM.

### C. The "Sensory" Attention (Feature Selection)
- **Role:** Identifying relevant historical context from `history.txt`.
- **PAL Ops:** `ai::attend`.
- **Logic:** Uses `attention_module.c` logic to weight previous user failures (e.g., specific functional groups) more heavily in the current context.
- **Recursion:** Attention scores are used as input "features" for the RL policy.

### D. Meta-FSM Control
A top-level PAL script monitors **Attention Weights**. If user engagement (response latency) drops, the Meta-FSM overrides the RL agent to switch from "Hard Mode" to "Engagement Mode" (easier questions) to prevent user dropout.

## 4. Cross-Project AI Synergy

### A. Testing Framework (Automated QA)
- Bots programmed via `op-ed` (PAL scripts) will "take" the quiz to verify the `quiz_orchestrator` logic.
- These bots use the same `bot_module.c` FSM engine as the `ai_tutor`.
- **KPI:** Regression pass rate of AI bots on the Question Bank.

### B. Games (LSR - Lunar Streetrace Raider)
- LSR NPCs (FuzzPets) use the Quiz-Bot as a **Validation Engine** for R&D.
- **Workflow:**
    1. `ai-labs` distills a corpus into a "Tech Ledger".
    2. The FuzzPet "studies" the ledger and takes a Quiz (via Quiz-Bot).
    3. The Quiz Score determines the quality/success of the generated tech recipe in LSR.

### C. Knowledge Distillation (`ai-labs`)
- Manual quiz results serve as the "Human Feedback" loop (RLHF) for local models.
- High-performing quiz paths are "distilled" into **Knowledge Ledgers** stored in `pieces/master_ledger/`.
