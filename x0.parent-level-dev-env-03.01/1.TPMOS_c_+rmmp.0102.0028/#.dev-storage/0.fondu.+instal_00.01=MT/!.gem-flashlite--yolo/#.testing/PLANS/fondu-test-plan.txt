PART 3: FULL VERIFICATION WORKFLOW (Run This After Implementing Fondu)
================================================================================

### Test Scenario 1: Fresh Install
```bash
# Setup
cd /path/to/TPMOS

# Step 1: Create test project
mkdir -p projects/test_fondu/{ops/+x,layouts,pieces}
echo "[META]" > projects/test_fondu/project.pdl
echo "project_id = test_fondu" >> projects/test_fondu/project.pdl
echo "entry_layout = projects/test_fondu/layouts/test.chtpm" >> projects/test_fondu/project.pdl

# Step 2: Create dummy op
echo '#include <stdio.h>' > projects/test_fondu/ops/dummy_op.c
echo 'int main() { printf("Dummy op executed\n"); return 0; }' >> projects/test_fondu/ops/dummy_op.c

# Step 3: Create ops manifest
echo 'dummy_op|Test dummy operation|none' > projects/test_fondu/ops/ops_manifest.txt

# Step 4: Install via Fondu
./fondu --install test_fondu

# Step 5: Verify
[ -d pieces/apps/installed/test_fondu/ ] && echo "✓ Install directory exists"
[ -f pieces/os/ops_registry/test_fondu.txt ] && echo "✓ Ops registry exists"
./fondu --list-ops | grep -q "dummy_op" && echo "✓ Op appears in catalog"
```

### Test Scenario 2: Uninstall
```bash
# Step 1: Uninstall
./fondu --uninstall test_fondu

# Step 2: Verify
[ ! -d pieces/apps/installed/test_fondu/ ] && echo "✓ Install directory removed"
[ ! -f pieces/os/ops_registry/test_fondu.txt ] && echo "✓ Ops registry removed"
./fondu --list-ops | grep -q "dummy_op" || echo "✓ Op removed from catalog"
[ -d projects/test_fondu/ ] && echo "✓ Source project still exists"
```

### Test Scenario 3: Archive & Restore
```bash
# Step 1: Archive
./fondu --archive test_fondu

# Step 2: Verify archived
[ -d projects/trunk/test_fondu/ ] && echo "✓ Project moved to trunk"
[ ! -d projects/test_fondu/ ] && echo "✓ Original project removed"
grep -q "test_fondu" pieces/os/compiled_projects.txt || echo "✓ Removed from compile list"

# Step 3: Restore
./fondu --restore test_fondu

# Step 4: Verify restored
[ -d projects/test_fondu/ ] && echo "✓ Project restored"
[ ! -d projects/trunk/test_fondu/ ] && echo "✓ Trunk copy removed"
grep -q "test_fondu" pieces/os/compiled_projects.txt && echo "✓ Added back to compile list"
```

### Test Scenario 4: PAL Script Integration
```bash
# Prerequisites
./fondu --install user
./fondu --install fuzz-op

# Create test PAL script in fuzz-op
echo 'OP user::create_profile "pal_test_user"' > projects/fuzz-op/test_script.pal

# Run PAL script (via prisc or fuzz-op manager)
./prisc projects/fuzz-op/test_script.pal

# Verify profile created
[ -f projects/user/pieces/profiles/pal_test_user/state.txt ] && echo "✓ PAL op call worked"
```

================================================================================
PART 4: SUCCESS CRITERIA SUMMARY
================================================================================

Fondu is considered WORKING when ALL of the following are true:

□ All 7 KPIs pass (install, uninstall, archive, restore, list, list-ops, PAL integration)
□ No critical system files are corrupted
□ Manifest files update cleanly (no duplicates, no corruption)
□ Project source files are never modified by Fondu
□ Installed binaries are read-only (chmod 555)
□ Ops registry accurately reflects installed projects
□ PAL scripts can call ops from other projects
□ compile_all.sh only compiles projects in compiled_projects.txt

Fondu is considered BROKEN if ANY of the following occur:

✗ System crashes during Fondu operations
✗ Files are corrupted or unreadable
✗ Projects disappear (source deleted when it shouldn't be)
✗ Ops registry doesn't match installed projects
✗ PAL scripts can't call external ops
✗ Installed apps don't launch
✗ Compile cycle breaks after Fondu operations
